# AI-Discussion-Process

A collaborative development methodology with AI to build lightweight, robust Python scripts, optimized even for Android and limited resource environments.

---

## 📌 概要 / Overview

本プロジェクトは、AIを単なる「コード生成ツール」として使うのではなく、**「対等な開発パートナー」**として深いディスカッション（対話）を重ね、現場実用に耐えうるシステムを削り出すための**【AI協調開発プロセス】**の実証・公開リポジトリです。

arrowsTAB（Android環境等）をはじめとする、限られたリソースや型落ちPCといった制約のある環境下でも、CPUに負荷をかけず「軽量・軽快・打たれ強い」Pythonスクリプトを構築するアプローチを定義しています。

---

## 🛠️ コア・フィロソフィー（開発哲学）

AIとの対話において、私たちが徹底している3つの原則です。

1. **ファスト思考の排除（Slow & Deep Discussion）**
   AIに対して「動くコードをすぐに出して」という即物的な要求（ファスト思考）をあえて排除します。あえて時間をかけてでも、設計の背景やリスク、エラーハンドリングについて議論を深め、解答の「精度」と「堅牢性」を極限まで高めます。

2. **現場主義のリスク管理（Robust Architecture）**
   20年のモノづくり現場で培った「段取りの良さ」と「トラブルを起こさない徹底したリスク管理」の視点をコード設計に持ち込みます。サイトのブロック対策（擬態化ロジック）や、切断時にも進捗を失わない追記型リアルタイムログなど、実用に耐えうる防衛線を標準実装します。

3. **リソースの最適化（Lightweight Design）**
   高価な最新マシンのパワーに頼るのではなく、クリーンで無駄のないコードを構築することで、バックグラウンドで静かに、かつサクサク動作する「マシンに優しいシステム」を削り出します。
